Introduction

A programming language is a set of instructions that allows communication between humans and computers. It is used to write programs that control the behavior of a machine and to express algorithms precisely. Programming languages vary in syntax and purpose, from high-level languages like Python and Java to low-level languages like Assembly. They facilitate the development of software, applications, websites, games, and much more. The choice of language depends on the project's needs and the programmer's preferences.

JavaScript and Java

JavaScript and Java are two popular programming languages, but they have different purposes and distinct characteristics.

JavaScript

  • Type: Scripting language.
  • Main Use: Web development, especially for adding interactivity to websites and web applications.
  • Execution: Interpreted by web browsers.
  • Syntax: Similar to other C-based programming languages.
  • Features: Supports DOM manipulation, events, AJAX, and more.
  • Popular Frameworks and Libraries: React, Angular, Vue.js, Node.js.

Java

  • Type: Object-oriented programming language.
  • Main Use: Enterprise applications, mobile applications (especially Android), embedded systems, and web applications.
  • Execution: Compiled to bytecode and run on the Java Virtual Machine (JVM).
  • Syntax: Strictly typed with a more robust syntax.
  • Features: Supports multithreading, automatic garbage collection, large standard library.
  • Popular Frameworks: Spring, Hibernate, JavaFX.

Key Differences:

  • Execution Environment: JavaScript is executed in the browser, while Java is executed on the JVM.
  • Typing: JavaScript is dynamically typed, while Java is statically typed.
  • Applications: JavaScript is mainly used for front-end web development, while Java is used for backend applications, mobile applications, and enterprise systems.

Variables

  • var: Declares a variable, optionally initializing it with a value. It has function scope.
  • let: Declares a block-scoped local variable, optionally initializing it with a value.
  • const: Declares a block-scoped constant, which must be initialized at the time of declaration.

Primitive types in JavaScript include:

  • number: Integer and floating-point numbers.
  • string: Strings of characters.
  • boolean: True or false.
  • undefined: Default value of uninitialized variables.
  • null: Represents intentional absence of value.
  • symbol: Immutable and unique data type.
  • bigint: For large integers.

Declaring variables
Using var:

var name = 'Alice';

Using let: let year = 2020;

Using const: const pi = 3.14159;

Variable scope

In JavaScript, variables can have different scopes depending on how they are declared:

  • Global Scope: Variables declared outside any function or block have global scope, accessible from any part of the code.
    • var globalVar = "I am available everywhere"
  • Function Scope: Variables declared inside a function using var are limited to that function's scope and are not accessible outside of it.
    • function localScope() { var localVar = "I am available only within this function"; console.log(localVar); // Accesses the local variable }
  • Block Scope: Variables declared with let or const inside a block (such as { ... }) are limited to that block's scope. This includes control structures like loops and conditionals.
    • if (true) { let blockLet = "I am available only within this block"; const blockConst = "I am also available only within this block"; console.log(blockLet); // Accesses the block-scoped variable console.log(blockConst); // Accesses the block-scoped variable }
  • Hoisting: var variable declarations are "hoisted" to the top of their scope, allowing them to be used before they are declared but not initialized. let and const variables are also hoisted but cannot be accessed before their declaration.
    • console.log(hoistedVar); // undefined, because the declaration was hoisted var hoistedVar = "I am hoisted but not initialized"; // console.log(hoistedLet); // This would cause an error because let and const cannot be accessed before declaration let hoistedLet = "I cannot be accessed before declaration";

Global variables

Global variables in JavaScript are those declared outside any function or block, making them accessible from any part of the code. They can be created with var, let, or const, but global variables are not recommended due to the risk of name conflicts and debugging difficulties. Excessive use of global variables can lead to less modular and harder-to-maintain code. It's a good practice to limit the scope of variables as much as possible to improve code readability and maintainability.

Constants

In JavaScript, constants are declared using the const keyword. A constant must be initialized at the time of declaration, and its value cannot be reassigned later. Constants have block scope, meaning they are accessible only within the block where they were defined. Although the value of a constant cannot be changed, if the value is an object or array, its properties or elements can be modified. Using const is ideal for values that should remain immutable throughout the code.

const favoriteColor = 'blue';
Data types

Primitive Types

  • Number: Represents numbers, including integers and floating-point numbers.
  • String: Sequences of characters used to represent text.
  • Boolean: Logical values that can be true or false.
  • Undefined: The default value for variables that have not been initialized.
  • Null: Represents an intentional absence of value.
  • Symbol: Unique and immutable values, used to uniquely identify object properties.
  • BigInt: Used to represent large integers beyond the limit of the Number type.

Complex Types

  • Objects: Structures that allow storing collections of data and more complex entities.
  • Arrays: Ordered lists of values that can be of different types.
  • Functions: Blocks of code that can be called and executed later.

If else

The if...else control flow in JavaScript is used to execute different blocks of code based on specific conditions.

Structure

if (number > 5) { console.log("The number is greater than 5"); } else { console.log("The number is 5 or less"); }

Multiple Conditions

let grade = 85; if (grade >= 90) { console.log("A"); } else if (grade >= 80) { console.log("B"); } else if (grade >= 70) { console.log("C"); } else { console.log("F"); }

Nested Conditions

let age = 20; let hasPermission = true; if (age >= 18) { if (hasPermission) { console.log("You may enter."); } else { console.log("Permission is required."); } } else { console.log("Entry is prohibited.");
While

The while loop in JavaScript is used to execute a block of code repeatedly while a specified condition is true. It is useful when you do not know beforehand how many times the block of code needs to be executed.

Syntax

let count = 0; while (count < 5) { console.log("Count is: " + count); count++; }

In this example, the code inside the while block will execute as long as the variable count is less than 5. With each iteration, the value of count is incremented by 1.

Function declaration

Functions in JavaScript are blocks of code designed to perform a specific task and can be executed whenever needed. They allow for code reuse, modularity, and improved readability. Functions can be declared using the function keyword or as arrow functions. They can take parameters and return values. Here's a basic example of a function declaration:

function greet(name) { return "Hello, " + name; } console.log(greet("Alice")); // Outputs: Hello, Alice
